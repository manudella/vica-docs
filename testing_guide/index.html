    VICA Testing Guide¶

    Prerequisites¶

    Hardware Note¶

    This setup assumes a single GPU (e.g. RTX 5070). The OCR model (Qwen2.5-VL via vLLM) and LLM (Ollama) cannot run simultaneously. The backend auto-orchestrates GPU switching:

    1. OCR phase — Qwen2.5-VL on the GPU (parse CVs and JD documents)
    2. LLM phase — Ollama on the GPU (skill extraction, JD skill extraction, match analysis)

    Software Setup¶

    Service Default URL Purpose
    Backend http://localhost:8000 FastAPI server
    Frontend http://localhost:5173 Vite React dev server
    OCR (Qwen2.5-VL) http://localhost:8001 Vision OCR for PDF/image parsing (vLLM)
    Ollama http://localhost:11434 LLM for skill extraction/analysis

    Start Order¶

    # Terminal 1: Backend (always running)
cd vica
python run_backend.py

# Terminal 2: Frontend (always running)
cd vica/frontend
npm run dev

    Prepare Test Files¶

    • 2-3 CV PDFs (real or sample)
    • 1 job description (can paste text directly)

    Test Procedure¶

    Phase A: Upload CVs (no GPU needed)¶

    1. Open http://localhost:5173
    2. Verify the stepper shows 4 steps: Upload CVs > Job Description > Validate Skills > Match & Reports
    3. Step 1 should be active (blue circle)
    4. Click "Upload CVs" — select 2-3 PDF files (multi-select supported)
    5. Verify all files appear in the table with gray "Uploaded" badges
    6. Progress bar shows "0/N ready"
    7. The "Next" button should be disabled (no candidates are ready yet)

    Phase B: Parse CVs (GPU: OCR auto-switch)¶

    1. The backend will automatically stop Ollama and start the OCR server (Qwen2.5-VL) when needed
    2. In the UI, check the GPU status badge — click "Refresh" if needed, should show "GPU: OCR active"
    3. Click "Parse All (OCR)"
    4. Wait for completion — status message shows "N CVs parsed"
    5. All candidates should now show yellow "Parsed" badges
    6. Progress bar updates

    Phase C: Extract Skills (GPU: LLM auto-switch)¶

    1. The backend will automatically stop the OCR server and load the Ollama model when needed
    2. Click the GPU "Refresh" button — should show "GPU: LLM active"
    3. Click "Extract Skills (LLM)"
    4. Wait for completion (this takes longer — LLM processes each CV)
    5. Status message shows "N skills extracted"
    6. All candidates show green "Ready" badges
    7. Progress bar shows "N/N ready"
    8. The "Next" button is now enabled
    9. Click "Next: Job Description"

    Phase D: Create Job Description (GPU: Ollama stays on)¶

    Keep Ollama running — the next steps need it for JD skill extraction.

    1. Stepper shows step 2 active
    2. Option A: Click "Upload JD (PDF/image)" if you have a JD file
    3. Note: this needs the OCR server — the backend will auto-switch GPU if needed. Or just paste text instead.
    4. Option B (recommended for testing): Paste job description text into the textarea
    5. Click "Create Job"
    6. Verify green badge shows "Job #1 created"
    7. Click "Next: Validate Skills"

    Phase E: Validate & Order Skills (GPU: Ollama)¶

    1. Stepper shows step 3 active
    2. Skills are auto-extracted from the JD by the LLM — wait for the list to appear
    3. Verify the skill list shows with:
    4. Drag handles on the left
    5. Checkboxes (checked = required)
    6. Green left border for required skills

    7. Delete (X) button on the right

    8. Test drag-and-drop: Drag a skill to a new position

    9. Test toggle: Uncheck a skill's "Required" checkbox
    10. Badge should change from "Required" (green) to "Preferred" (gray)
    11. Left border changes from green to gray
    12. Test delete: Click the X button on a skill — it should disappear
    13. Verify the header counts update (e.g., "5 required, 2 preferred")
    14. Click "Save Skills" — verify "Saved" badge appears
    15. Click "Next: Match & Reports"

    Phase F: Match & Reports (GPU: Ollama)¶

    1. Stepper shows step 4 active
    2. Set "Top candidates" to 2 or 3
    3. Click "Run Match"
    4. Spinner shows "Ranking & Analyzing..." — this runs SPEA2 + LLM analysis
    5. Wait for results table to appear
    6. Verify the results show:
    7. Run number badge
    8. "N top" badge (number of analyzed candidates)
    9. "N total" badge (total candidates in the pool, if more than top)
    10. "N analyzed" badge

    11. Table with Rank, Candidate ID, R (SPEA2), Fitness score, Report buttons

    12. Test inline report: Click "Report" on a top candidate

    13. An iframe should expand below the row showing the full report

    14. Click "Hide" to collapse it

    15. Test new tab: Click the arrow button — report opens in a new browser tab

    16. Test remaining candidates list: Below the top results table, click "Show remaining N candidates"

    17. An expandable scrollable list appears with Rank, Candidate ID, R (SPEA2), Fitness, and CV button
    18. These candidates have no report (only top-K are analyzed)
    19. Click "Hide remaining N candidates" to collapse the list
    20. The list has a max height of 320px and scrolls vertically

    Phase G: Verify Report Content¶

    In the report (inline or new tab), check:

    Check Expected
    Fit score NOT present — no "X/10" numerical score anywhere
    Recommendation Badge visible: Strong Yes / Yes / Maybe / No
    Overall fit 1-2 sentence assessment text
    Strengths Bulleted list (green)
    Risks / Gaps Bulleted list (red)
    Required Skills table Each skill shows Status (check/cross), Match level badge, Evidence
    CV Citations label Each skill with evidence shows "CV Citations:" header
    Full citations ALL literal quotes from the CV shown (not truncated to 2-3)
    Preferred Skills table Same structure as required
    Transferable Skills Card layout with skill name, relation, reasoning
    Interview Questions Each question has a category badge: blue "Technical" or orange "Verification"
    Question types Mix of verification (probing weak claims) and technical depth (architecture, trade-offs)
    Priority badges Each question shows priority: high/medium/low
    Additional Skills Skills not in JD requirements listed with proficiency
    Candidate Profile Years of experience, education level, field of study (when extracted)

    Phase H: Navigation Tests¶

    1. Click "Back" from step 4 to step 3 — state should be preserved
    2. Click "Back" from step 3 to step 2 — job description text still there
    3. Click "Back" from step 2 to step 1 — candidates still listed and ready
    4. Click a completed step circle in the stepper — should navigate there directly
    5. Try clicking a future step — should not navigate unless current step is complete

    Phase I: Sidebar & Settings Tests¶

    1. Sidebar shows Jobs, Candidates, Validation Runs navigation links with counts
    2. Sidebar shows Quick Actions: New Job, New Validation Run
    3. Sidebar shows Overview stats: jobs ready, total candidates, validations done
    4. Click Settings in sidebar — navigates to Settings page
    5. Settings page shows GPU Management with activate/release controls
    6. Settings page shows LLM Configuration (model, URL, connectivity status)
    7. Settings page shows OCR Configuration (URL, health)
    8. Settings page shows About section with version info

    Phase J: Search & Filter Tests¶

    1. On Jobs page, type in search bar — filters jobs by title in real-time
    2. Use status dropdown to filter "Skills ready" vs "Needs setup"
    3. Click sort buttons (Date, Title, CVs) — list reorders; click again to toggle asc/desc
    4. "Showing X of Y jobs" counter appears when filters are active
    5. On Validation Runs page, search by title — filters runs in real-time
    6. Use type filter (Historical / Supervised) and status filter (Setup, Completed, etc.)
    7. Sort buttons (Date, Title, Status) work correctly
    8. Clear search and reset filters — full list reappears

    Edge Cases¶

    No GPU services running¶

    1. Stop both the OCR server and Ollama
    2. Upload a .txt file as CV — should parse without OCR (text files don't need it)
    3. GPU badge shows "GPU: no service detected"
    4. Helpful hints appear explaining which service to start

    LLM disabled¶

    1. Set LLM_ENABLED=false in .env or environment
    2. Restart backend
    3. Skill extraction uses phrase matching fallback (no GPU needed)
    4. Match analysis uses fallback mode (no LLM analysis, basic binary matching)
    5. Report still renders but with limited data

    Empty states¶

    1. No candidates uploaded — step 1 shows "No candidates yet" message
    2. Run match with 0 candidates — should show clear error message
    3. JD with no extractable skills — skill editor shows "No skills extracted" message

    API Endpoints Reference (for manual testing)¶

    # GPU status
curl http://localhost:8000/gpu-status

# Upload a CV
curl -X POST http://localhost:8000/candidates/upload -F "file=@cv.pdf"

# Batch parse (OCR phase)
curl -X POST http://localhost:8000/candidates/batch-parse

# Batch extract skills (LLM phase)
curl -X POST http://localhost:8000/candidates/batch-extract

# Create job
curl -X POST http://localhost:8000/jobs/ \
  -H "Content-Type: application/json" \
  -d '{"raw_text": "Job description text here..."}'

# Extract skills from JD
curl http://localhost:8000/jobs/extract-skills?job_id=1

# Save skill order
curl -X PUT http://localhost:8000/jobs/1/skills \
  -H "Content-Type: application/json" \
  -d '{"skills": [{"name": "Python", "position": 0, "required": true}]}'

# Run match + analyze
curl -X POST http://localhost:8000/match/run-and-analyze \
  -H "Content-Type: application/json" \
  -d '{"job_id": 1, "archive_size": 5}'

# View report
curl http://localhost:8000/report/cv/1/1

    Troubleshooting¶

    Problem Solution
    "Parse All" fails Check OCR server is running: curl http://localhost:8001/health (backend auto-starts it, check logs)
    "Extract Skills" fails Check Ollama is running: curl http://localhost:11434/v1/models
    Both services needed at once Not possible with single GPU. The backend auto-switches between OCR and LLM as needed
    Skill extraction returns 0 skills Check Ollama has the model loaded: ollama list should show llama3.2
    Report shows "No evidence found" LLM may not have run analysis — check that "Run Match" completed with "N analyzed"
    JD skill extraction empty Ensure the job description text is substantial enough for the LLM to extract skills
    Frontend not connecting Check Vite proxy config or CORS — backend should be on port 8000
