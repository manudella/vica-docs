    Regular Hiring Loop — Step-by-Step Guide¶

    What is VICA?¶

    VICA is a tool that helps you screen candidates for a job opening. You upload CVs and a job description, and VICA reads each CV, identifies skills, and ranks candidates based on how well their skills match the job requirements. It does not make hiring decisions — it gives you a ranked shortlist with evidence to support your review.

    Before You Start¶

    • The system should be running (you'll see the VICA interface at http://localhost:3000 or your configured URL)
    • Have your job description ready (text you can paste, or a PDF/image file)
    • Have your CV files ready (PDF, DOCX, or image files)
    • Check the GPU status badge in the sidebar footer — it shows whether the AI services are available
    • For detailed GPU and system configuration, click Settings at the bottom of the sidebar

    Step 1: Create a Job¶

    1. From the left sidebar, click Jobs (or use the New Job quick action in the sidebar)
    2. Click "New Job" in the page header to open the creation form
    3. You have two options:
    4. Paste text: Copy your job description into the text area
    5. Upload PDF/image: Click "Upload JD" to upload a PDF or image of the JD — the system will extract the text using OCR
    6. Give the job a title (e.g., "Senior Backend Engineer")
    7. Click "Create Job"
    8. A green badge confirms the job was created

    Step 2: Upload CVs¶

    1. Navigate to the job detail view (click on the job you just created)
    2. Go to the Candidates tab
    3. Click "Upload CVs" — you can select multiple files at once
    4. Each uploaded file appears in the table with a gray "Uploaded" badge
    5. The progress bar shows how many candidates are ready (0 at this point)

    Supported formats: PDF, DOCX, PNG, JPG, WEBP, TXT

    Step 3: Parse CVs¶

    1. Click "Parse All (OCR)"
    2. The system automatically activates the OCR service (you may see the GPU badge change to "OCR active")
    3. Wait for parsing to complete — the status message shows progress
    4. All candidates now show yellow "Parsed" badges
    5. At this point, VICA has read the text from each CV and removed personal information (names, emails, phone numbers)

    What if parsing fails? Check that the backend is running. For non-PDF text files, OCR is not needed.

    Step 4: Extract Skills¶

    1. Click "Extract Skills (LLM)"
    2. The system automatically switches to the AI model (GPU badge changes to "LLM active")
    3. This step takes longer — the AI reads each CV individually and identifies skills with evidence
    4. When complete, candidates show green "Ready" badges
    5. The progress bar shows "N/N ready"
    6. The "Next" button is now enabled

    What happens here: For each CV, the AI identifies every professional and technical skill, how proficient the candidate appears to be, and the exact CV text that supports each skill claim.

    Step 5: Review & Validate Job Skills¶

    1. Click "Next" to go to the Description & Skills tab
    2. VICA's AI automatically extracts skills from your job description
    3. You'll see a list of skills with:
    4. Drag handles on the left — drag to reorder by importance (most important at top)
    5. Checkboxes — checked = Required (green border), unchecked = Preferred (gray border)
    6. Delete (X) button — remove skills that aren't relevant
    7. Review the list carefully:
    8. Are the most important skills at the top?
    9. Are the right skills marked as "Required" vs "Preferred"?
    10. Are there any irrelevant skills to remove?
    11. Click "Save Skills" — a "Saved" badge confirms

    This step matters: The order and required/preferred classification directly affect how candidates are ranked. Skills at the top of the list carry more weight.

    Step 6: Run Match¶

    1. Click "Next" to go to the Match & Reports tab
    2. Set the "Top candidates" number (how many candidates you want shortlisted)
    3. Click "Run Match"
    4. A spinner shows progress:
    5. First: "Ranking..." — the algorithm scores all candidates
    6. Then: "Analyzing candidate 1/N..." — the AI writes a detailed analysis for each shortlisted candidate
    7. When complete, you'll see a results table with:
    8. Rank — position in the shortlist
    9. Candidate — filename or ID
    10. Fitness score — how well they match (higher = better match)
    11. Report buttons — view detailed analysis

    Step 7: Read Reports¶

    1. Click "View Report" on any candidate to see the detailed report inline
    2. Click the arrow icon to open the report in a new tab
    3. Each report contains:
    4. Recommendation badge — Strong Yes / Yes / Maybe / No
    5. Overall fit — 1-2 sentence summary
    6. Strengths and risks — bullet-point highlights
    7. Required skills table — each skill with match evidence and CV citations
    8. Preferred skills table — same format
    9. Transferable skills — relevant skills not in the job requirements
    10. Interview questions — targeted questions to ask this candidate

    For details on how to interpret each report section, see Understanding Reports.

    Tips & FAQ¶

    Q: What if I disagree with the ranking? Go back to Step 5 and adjust the skill order or required/preferred toggles. Rerun the match — results will change based on your adjustments.

    Q: What if no skills are extracted from a CV? The CV text might be too short, in an unusual format, or the AI couldn't parse it. Check that parsing completed (yellow "Parsed" badge). If the CV is an image-only PDF, ensure OCR is working.

    Q: Can I add more candidates after running a match? Yes. Upload and process new CVs, then rerun the match.

    Q: What if the AI service is unavailable? VICA falls back to keyword-based skill matching (less accurate but still functional). The GPU status badge will indicate when services are not available.

    Q: How long does it take? - Parsing: a few seconds per CV - Skill extraction: 15-60 seconds per CV (depends on length and GPU) - Matching + analysis: 30-90 seconds per shortlisted candidate

    For detailed UI testing steps, see the Testing Guide.
