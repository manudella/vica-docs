    Validation Metrics Reference¶

    Mathematical definitions and interpretation of all statistics computed by _compute_validation_stats() in app/api/validation.py:61.

    Ground Truth Definition¶

    Historical validation¶

    • Positive set: Candidates marked as "interviewed" or "hired" via PUT /jobs/{job_id}/candidates/{cid}/status
    • Top-K determination: K = count of interviewed candidates. If zero, K = total ranked candidates.

    Supervised validation¶

    • Positive set: Candidate IDs in recruiter_picks (set via POST /validation/{id}/recruiter-picks)
    • Top-K determination: K = recruiter_pick_count (count of recruiter picks). If zero, K = total ranked candidates.

    Core Metrics¶

    Sets¶

    model_top_k         = set of first K candidate IDs in SPEA2 ranking
ground_truth_positives = set of positive candidate IDs (interviewed/hired or recruiter picks)

TP (True Positives)  = model_top_k ∩ ground_truth_positives
FP (False Positives) = model_top_k - ground_truth_positives
FN (False Negatives) = ground_truth_positives - model_top_k

    Precision¶

    Precision = TP / (TP + FP)

    "Of the candidates VICA selected in its top-K, what fraction are actual positives?"

    High precision = VICA's picks are reliable. Low precision = VICA recommends too many wrong candidates.

    Recall¶

    Recall = TP / (TP + FN)

    "Of the actual positive candidates, what fraction did VICA place in its top-K?"

    High recall = VICA finds most of the right candidates. Low recall = VICA misses candidates you care about.

    F1 Score¶

    F1 = 2 × Precision × Recall / (Precision + Recall)

    Harmonic mean of precision and recall. Ranges from 0 to 1. F1 = 0 when either precision or recall is 0. F1 = 1 when both are perfect.

    Accuracy@K¶

    Accuracy@K = TP / K

    "What fraction of VICA's top-K slots contain actual positives?" Equivalent to precision when K equals the model's top-K size.

    Rank Analysis¶

    For each ground-truth positive candidate, find their position in VICA's full ranking (1-indexed).

    Metric Definition
    avg_rank Mean rank position of ground-truth positives
    median_rank Median rank position
    best_rank Lowest (best) rank among positives
    worst_rank Highest (worst) rank among positives
    not_ranked Count of positives not in the ranking at all
    positions Sorted list of all rank positions

    Interpretation: If ground-truth candidates cluster near the top (low rank numbers), VICA's ordering agrees with reality. If avg_rank is much larger than K, VICA is placing the right candidates too low.

    Skill Coverage¶

    Per-skill analysis: how many candidates in the pool have each job skill.

    {
  "skill": "Python",
  "candidates_with_skill": 18,
  "total_candidates": 50,
  "coverage_pct": 36.0
}

    Interpretation: - Skills with very high coverage (>80%) are not discriminating — almost everyone has them - Skills with very low coverage (<10%) might be too niche or poorly matched by the taxonomy - Useful for identifying which skills actually differentiate candidates in the pool

    Fitness Distribution¶

    Statistical summary of SPEA2 fitness scores across all candidates.

    Overall¶

    {
  "mean": 3.2500,
  "median": 3.0000,
  "std": 1.1500,
  "min": 0.2000,
  "max": 6.8000
}

    Ground-truth positives vs non-positives¶

    {
  "ground_truth_fitness": {"mean": 4.8000, "median": 5.0000},
  "non_ground_truth_fitness": {"mean": 2.9000, "median": 2.7000}
}

    Interpretation: If ground-truth candidates have significantly higher mean fitness than non-positives, VICA's scoring function is working well — it assigns higher scores to the "right" candidates even if the top-K cutoff doesn't capture all of them.

    Historical-Specific Metrics¶

    Only computed for historical validation:

    Metric Definition
    interviewed_count Total candidates marked as interviewed
    hired_count Total candidates marked as hired
    hired_in_top_k Count of hired candidates that appear in VICA's top-K

    Aggregate Report¶

    Computed across all completed validation runs (GET /validation/aggregate/report).

    {
  "total_runs": 5,
  "historical_runs": 3,
  "supervised_runs": 2,
  "avg_precision": 0.7200,
  "avg_recall": 0.6500,
  "avg_f1": 0.6833,
  "avg_accuracy_at_k": 0.7000,
  "std_precision": 0.0800,
  "std_recall": 0.1200,
  "std_f1": 0.0950,
  "total_candidates_evaluated": 250,
  "total_ground_truth": 40,
  "runs": [...]
}

    Standard deviation is only computed when there are 2+ runs. Low std = consistent performance. High std = variable results across runs (may indicate sensitivity to skill configuration).
