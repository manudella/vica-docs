    AI & ML Engineering¶

    Technical documentation for VICA's AI pipeline — from skill extraction through multi-objective ranking to semantic analysis.

    Pipeline Overview¶

    graph TB
    subgraph Ingestion["1. Document Ingestion"]
        OCR["OCR (Qwen2.5-VL-3B)<br/>PDF/Image → Text"]
        PII["PII Anonymization<br/>Presidio + spaCy"]
        OCR --> PII
    end

    subgraph Extraction["2. Skill Extraction"]
        DEEP["Deep LLM Extraction<br/>llama3.1:8b"]
        NORM["Taxonomy Normalization<br/>ESCO + RapidFuzz"]
        VEC["Binary Skill Vector<br/>has/doesn't have"]
        DEEP --> NORM --> VEC
    end

    subgraph Ranking["3. Multi-Objective Ranking"]
        MATRIX["Candidate × Skill Matrix<br/>Polars/Pandas"]
        SPEA["SPEA2<br/>Pareto Dominance + Strength"]
        MATRIX --> SPEA
    end

    subgraph Analysis["4. Semantic Analysis"]
        MATCH["Per-Candidate LLM Analysis<br/>Match levels, evidence, questions"]
        REPORT["Report Generation<br/>Jinja2 templates"]
        MATCH --> REPORT
    end

    Ingestion --> Extraction --> Ranking --> Analysis

    Scoring Flow¶

    graph LR
    CV["CV Text"] --> DE["Deep Extraction"]
    DE --> PROF["Proficiency<br/>expert/proficient/<br/>familiar/mentioned"]
    DE --> JUST["Justification<br/>strong/moderate/<br/>weak/unsubstantiated"]
    DE --> EVID["Evidence<br/>Verbatim CV quotes"]
    DE --> BIN["Binary Vector<br/>1 or 0"]

    BIN --> SPEA2["SPEA2 Ranking<br/>(binary × weights)"]

    SPEA2 --> SEM["Semantic Analysis"]
    SEM --> ML["Match Level<br/>strong/partial/<br/>transferable/no_match"]
    SEM --> REC["Recommendation<br/>strong_yes/yes/<br/>maybe/no"]
    SEM --> IQ["Interview Questions<br/>verification + depth"]

    style BIN fill:#fff3e0,stroke:#ef6c00
    style SPEA2 fill:#fff3e0,stroke:#ef6c00

    SPEA2 uses binary skill presence only

    Proficiency, justification, and evidence are not used in ranking — they exist for reports and post-match analysis.
