    VICA Production Deployment — Cost Estimation and ROI Analysis¶

    Investment analysis for deploying VICA in a medium-big consulting company. Target environment: 500-5,000 employees, 50-200 recruiters, 5,000-20,000 CVs processed per year. All figures in USD. Estimates as of Q1 2026.

    1. Executive Summary¶

    Investment Overview - Year 1 total investment: $115,000 - $265,000 (hardware + development + deployment) - Annual recurring cost (Years 2-5): $33,000 - $75,000 - 5-Year TCO: $247,000 - $565,000 - Comparable cloud solution 5-Year TCO: $750,000 - \(2,000,000 - **ROI**: 150-400% over 5 years - **Break-even**: 8-14 months from go-live - **Recruiter hours saved**: 12,500 - 75,000 over 5 years - **Cost per CV screen**: <\)8 (vs \(25-\)40 manual)

    Consulting firms screen thousands of CVs per year for both project staffing and external recruitment. Whether matching internal consultants to client engagements or evaluating external candidates for open positions, the screening process is labor-intensive and time-sensitive. Manual screening costs 24-45 minutes per CV when done thoroughly — reading the document, identifying relevant skills, comparing against job requirements, documenting the assessment, and preparing interview questions. At scale, this represents a significant portion of recruiter capacity that could be directed toward higher-value activities such as candidate engagement, client relationship management, and strategic workforce planning.

    VICA reduces the active recruiter time per CV from 24-45 minutes to 1-3 minutes by automating the read-extract-match-document pipeline. The system processes CVs through local OCR, extracts skills with proficiency levels using a local LLM, matches candidates to job requirements using a multi-objective evolutionary algorithm (SPEA2), and generates citation-backed evidence reports with interview questions. Crucially, every skill assessment is linked to verbatim quotes from the original CV — a level of traceability that manual screening cannot match consistently, especially under time pressure.

    The open-source stack eliminates recurring software license fees entirely. VICA runs on-premise using free and open-source components (FastAPI, React, Ollama, vLLM, Docker), meaning the primary investment is human work for integration, customization, and ongoing maintenance. This front-loaded cost structure contrasts sharply with cloud AI screening platforms that charge \(150,000-\)400,000 per year in recurring license fees. Over a 5-year horizon, VICA delivers 47-83% lower total cost of ownership while keeping all candidate data on-premise — a material advantage for GDPR compliance and EU AI Act readiness.

    Hardware Justification¶

    VICA's GPU orchestrator automatically switches between the OCR service (Qwen2.5-VL-3B-Instruct on vLLM, ~6GB VRAM) and the LLM service (llama3.1:8b on Ollama, ~8GB VRAM) via Docker API calls. Both models fit comfortably within 24GB VRAM, making the RTX 4090 a cost-effective entry point.

    Tier 1 (RTX 4090) handles up to approximately 25 CVs/hour with sequential GPU switching. Processing is serialized: OCR runs first (~30s/page), then the GPU switches to the LLM for skill extraction (~30-60s per candidate). This is sufficient for 5,000 CVs/year with batch processing during business hours.

    Tier 2 (A100 40GB) opens the door to significantly larger models. The llama3.1:70b model (~40GB VRAM) delivers an estimated 30-40% improvement in extraction accuracy over the 8b variant — particularly for nuanced skill identification, ambiguous CV formatting, and multilingual documents. Alternatively, the A100 can run both OCR and LLM models simultaneously (eliminating the GPU switching overhead entirely) if using the smaller models.

    Tier 3 (dual A100 80GB or H100 80GB) enables running OCR and LLM on separate GPUs simultaneously, effectively doubling throughput. This tier also supports frontier-class models such as Qwen2.5-VL-72B for OCR or DeepSeek-V3 for extraction, pushing quality toward cloud API parity while maintaining full on-premise control.

    Memory: VICA's in-memory store with JSON file persistence is lightweight. 64GB RAM handles approximately 50,000 candidate records comfortably. 128GB+ allows headroom for growth, OS caching, and concurrent processing buffers.

    Storage: The average CV PDF is approximately 200KB. Processing 20,000 CVs/year generates roughly 4GB/year of document storage. AI models are cached in Docker volumes — approximately 15GB total for the current model stack (Qwen2.5-VL-3B + llama3.1:8b), scaling to approximately 80GB for 70B-class models. NVMe storage ensures fast model loading on GPU switch events.

    2.4 Redundancy and Backup¶

    • NAS or secondary server for nightly backup of the data/ directory: $1,000 - $3,000

    • VICA's portable/CPU-only mode as emergency fallback (degraded performance, no GPU required)

    GPU hardware failure Low High — system downtime Warranty, rapid replacement agreement (48hr); VICA portable/CPU mode as degraded fallback
    Model quality insufficient for niche roles Low-Medium Medium — poor matching for some positions Upgrade to larger models (70B class); custom fine-tuning; supplement with recruiter review

    Recruiter adoption resistance Medium Medium — low utilization, poor ROI Phased rollout with champion users; clear ROI dashboards; recruiter feedback integration
    Integration complexity overrun Medium Medium — timeline slip, budget overrun Agile sprints, MVP-first approach, 15% contingency budget, scope management
    Regulatory change beyond current compliance Low Medium — unexpected requirements Open-source architecture enables rapid adaptation; active monitoring of EU AI Act guidance
    Scaling beyond single server capacity Low Low-Medium — performance degradation Multi-GPU upgrade path, horizontal scaling roadmap, queue-based processing
    Security breach of on-premise server Low High — data exposure, reputation Server hardening, network segmentation, access controls, encryption at rest, regular audits
    Open-source model license changes Low Low — need to switch models VICA's model-agnostic architecture (Ollama/vLLM) supports any compatible model

    Success Criteria for Go-Live¶

    • System processes 95%+ of uploaded CVs without errors
    • F1 score exceeds 0.65 in validation against historical hires
    • System handles peak load (50+ concurrent CVs) without degradation
    • Recruiter satisfaction survey averages above 4.0/5.0 in pilot

    • Monitoring and alerting in place (GPU utilization, system health, error rates)
    • Backup and disaster recovery tested
