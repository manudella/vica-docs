    Technology Stack¶

    Layer Technology
    Backend Python 3.11+, FastAPI, Uvicorn, Pydantic Settings
    Frontend React 18, TypeScript, Vite, @dnd-kit (drag-drop)
    LLM Ollama (llama3.1:8b), OpenAI-compatible API
    OCR vLLM (Qwen2.5-VL-3B-Instruct), vision model
    Data In-memory dict + JSON file persistence (no database)
    Matrix ops Polars, Pandas, NumPy
    Fuzzy matching RapidFuzz
    Agenderization Regex-based gender-neutral language replacement
    PDF parsing PyMuPDF (image rendering), pdfplumber (text extraction)
    Reports Jinja2 templates, WeasyPrint (optional PDF)
    Containerization Docker, Docker Compose, Nginx

    Service Layer¶

    Each service in app/services/ is a focused module with minimal coupling.

    Service File Responsibility Key dependencies
    CV Parser cv_parser.py Parse PDF/DOCX/image/text files into raw text ocr.py, pdfplumber
    OCR ocr.py Vision OCR via Qwen2.5-VL (PDF pages → text) httpx, PyMuPDF
    Anonymize anonymize.py Agenderize (gender-neutral language) + section CV Regex
    Skill Extraction skill_extraction.py Build binary skill vector with evidence llm_client.py, skill_library.py
    JD Extraction jd_extraction.py Extract + normalize skills from job descriptions llm_client.py, skill_library.py
    Taxonomy taxonomy.py ESCO skill resolution, fuzzy match, hierarchy RapidFuzz
    Skill Library skill_library.py SkillNormalizer wrapper around TaxonomyResolver taxonomy.py
    Skill Matrix skill_matrix.py Batch build candidate×skill matrix, persist skills Polars/Pandas, Store
    Skill Matching skill_matching.py Post-SPEA2 semantic analysis per candidate llm_client.py
    SPEA2 spea2.py Pareto dominance ranking algorithm NumPy
    Temporal temporal_extraction.py Extract years of experience + education via LLM llm_client.py
    Report report.py Generate HTML reports (Jinja2 templates) Jinja2
    GPU Orchestrator gpu_orchestrator.py Single-GPU switching between OCR and LLM Docker CLI, httpx
    LLM Client llm_client.py OpenAI-compatible API wrapper (all LLM calls) httpx

    Data Flow: Regular Loop¶

    graph TD
    A["1. Upload CV<br/>POST /candidates/upload"] -->|"Store file, create record"| B["2. Parse CV<br/>POST /candidates/{id}/preprocess"]
    B -->|"GPU → OCR, extract text, agenderize"| C["3. Extract Skills<br/>POST /candidates/{id}/extract-skills"]
    C -->|"GPU → LLM, deep extraction + taxonomy"| D["4. Create Job<br/>POST /jobs/"]
    D -->|"Store raw JD text"| E["5. Extract JD Skills<br/>GET /jobs/extract-skills"]
    E -->|"GPU → LLM, decompose + normalize"| F["6. Validate Skills<br/>PUT /jobs/{id}/skills"]
    F -->|"Recruiter reorders, toggles required"| G["7. Run Match<br/>POST /match/run-and-analyze (SSE)"]
    G -->|"SPEA2 ranking + LLM analysis"| H["8. View Report<br/>GET /report/cv/{id}/{job_id}"]

    Data Flow: Validation Loop¶

    Same as regular loop with these additions:

    graph TD
    A["1. Create Validation Run<br/>POST /validation"] -->|"historical or supervised"| B["2. Setup<br/>Upload CVs → Parse → Extract → Set JD → Validate"]
    B --> C{"Validation Type?"}
    C -->|Historical| D["3a. Mark Status<br/>interviewed / hired / rejected"]
    C -->|Supervised| E["3b. Set Recruiter Picks<br/>POST /validation/{id}/recruiter-picks"]
    D --> F["4. Run Match<br/>SPEA2 + LLM Analysis (SSE)"]
    E --> F
    F --> G["5. Compute Stats<br/>Precision, Recall, F1, Accuracy@K"]
    G --> H["6. Stats Report<br/>HTML per-run report"]
    H --> I["7. Aggregate Report<br/>HTML across all runs"]

    API Layer¶

    Five routers mounted in app/main.py:

    Router Prefix File Responsibility
    Candidates /candidates app/api/candidates.py Upload, parse, extract skills
    Jobs /jobs app/api/jobs.py CRUD, JD parsing, skill validation, candidate association
    Match /match app/api/match.py SPEA2 ranking, semantic analysis, SSE streaming
    Report /report app/api/report.py HTML report generation
    Validation /validation app/api/validation.py Validation runs, stats, recruiter picks

    Root endpoints in app/main.py: /health, /gpu-status, /gpu/ensure-ocr, /gpu/ensure-llm, /gpu/release, /llm-status.

    See API Reference for complete endpoint documentation.

    Frontend Structure¶

    frontend/src/
├── main.tsx               # React entry point
├── App.tsx                # Router: jobs | job-detail | global-candidates | validation
├── Sidebar.tsx            # Navigation sidebar
├── JobList.tsx            # List all jobs with counts
├── JobDetail.tsx          # 3-tab view: Candidates, Description & Skills, Match Results
│   ├── CandidatePool      # (embedded) Upload, parse, extract, status management
│   ├── JobSkillsEditor    # (embedded) Drag-drop skill ordering, required/preferred
│   └── MatchRun           # (embedded) Run SPEA2, view results + inline reports
├── GlobalCandidates.tsx   # All candidates across all jobs
├── ValidationRunList.tsx  # List validation runs
├── ValidationRunDetail.tsx # Full validation workflow (both historical + supervised)
└── CVManager.tsx          # Standalone candidate management

    Key patterns: - State management via useState + useCallback (no external state library) - SSE streaming for match progress (EventSource API) - Drag-and-drop via @dnd-kit (core + sortable) - All API calls go through /api proxy (Vite dev or Nginx production)
