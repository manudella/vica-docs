    Data Model & Persistence¶

    Architecture¶

    VICA uses an in-memory singleton store with JSON file persistence. No external database is required.

    • Speed: All reads are dict lookups (O(1))
    • Durability: Every mutation writes through to data/db/*.json via atomic temp-file-then-rename
    • Thread safety: ID counter uses threading.Lock; GPU orchestrator uses asyncio.Lock
    • Source: app/store.py

    Entities¶

    Skill¶

    @dataclass
class Skill:
    id: int
    name: str                # Original name as extracted
    normalized_name: str     # Lowercased canonical (used for dedup)
    source: str = "tmc"      # "esco" (taxonomy match) or "other" (open vocabulary)

    Candidate¶

    @dataclass
class Candidate:
    id: int
    external_id: str | None        # UUID assigned at upload
    cv_storage_key: str             # Path in data/files/ (e.g., "cvs/{uuid}_{filename}")
    cv_filename: str | None         # Original filename
    anonymized_text: str | None     # PII-removed CV text (after preprocess)
    evidence_index: str | None      # JSON: {skill_name: [quote1, quote2, ...]}
    temporal_features: str | None   # JSON: {years_of_experience: 8.0, ...}
    deep_skills: str | None         # JSON: [{name, proficiency, evidence, context, justification}, ...]
    match_analysis: str | None      # JSON: {skill_assessments, transferable_skills, interview_questions, summary}
    extraction_meta: str | None     # JSON: {extraction_method, model, skills_found, llm_enabled}
    analysis_meta: str | None       # JSON: {analysis_method, model}

    JobDescription¶

    @dataclass
class JobDescription:
    id: int
    external_id: str | None
    title: str | None
    raw_text: str                   # Full JD text
    created_at: str | None          # ISO 8601 UTC timestamp

    JobSkill¶

    @dataclass
class JobSkill:
    job_id: int
    skill_id: int
    position: int          # Recruiter-ordered position (0 = most important)
    required: bool = True  # True = required, False = preferred

    MatchRun¶

    @dataclass
class MatchRun:
    id: int
    job_id: int
    archive_size: int
    results: List[Tuple[int, float, float]]  # [(candidate_id, fitness, r_stat), ...]
    created_at: str | None

    ValidationRun¶

    @dataclass
class ValidationRun:
    id: int
    title: str
    run_type: str           # "historical" or "supervised"
    status: str             # "setup" → "candidates_ready" → "jd_ready" → "processing" → "matched" → "completed"
    created_at: str | None
    job_id: int | None              # Internal job created for this validation
    interviewed_count: int | None   # Historical: count of interviewed candidates
    recruiter_pick_count: int | None # Supervised: count of recruiter picks
    recruiter_picks: str | None     # JSON: [candidate_id, ...]
    stats: str | None               # JSON: full validation statistics blob
    match_run_id: int | None        # Reference to SPEA2 results

    candidate_skill: Set[Tuple[int, int]] — sparse representation. Only stores (candidate_id, skill_id) pairs for skills the candidate has.

    job_candidates: Dict[int, List[int]] — job_id → list of candidate_ids.

    job_skill: List[JobSkill] — ordered list; position determines weight in SPEA2.

    File Storage¶

    Directory: data/files/

    Key format: cvs/{uuid}_{original_filename}

    Interface: app/storage/base.py — LocalStorage class with upload(), download(), exists() methods.

    Persistence Files¶

    All in data/db/:

    File Contents Format
    counter.json {"next_id": N} Auto-increment counter
    skills.json {"1": {id, name, normalized_name, source}, ...} ID → Skill
    candidates.json {"1": {id, external_id, cv_storage_key, ...}, ...} ID → Candidate
    candidate_skill.json {"1": [3, 5, 12], "2": [1, 5]} candidate_id → [skill_ids]
    jobs.json {"1": {id, title, raw_text, created_at}, ...} ID → Job
    job_skill.json [{job_id, skill_id, position, required}, ...] Ordered list
    job_candidates.json {"1": [1, 2, 3], "2": [4, 5]} job_id → [candidate_ids]
    match_runs.json {"1": {id, job_id, archive_size, results, created_at}} ID → MatchRun
    candidate_status.json {"1": {"5": {"status": "hired", "updated_at": "..."}}} job_id → cand_id → status
    validation_runs.json {"1": {id, title, run_type, status, ...}} ID → ValidationRun

    Write strategy: Atomic writes via _atomic_write() — write to .tmp file, then os.replace() to target path. This prevents corruption if the process crashes mid-write.

    Singleton Access¶

    from app.store import get_store

store = get_store()  # Returns the singleton Store instance

    reset_store() clears all in-memory data and rewrites all JSON files with empty content. Used by test scripts.
