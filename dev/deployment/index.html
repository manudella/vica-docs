    Deployment Guide¶

    Local Development¶

    Prerequisites¶

    • Python 3.11+
    • Node.js 18+ (for frontend)
    • Ollama (for LLM) — https://ollama.com
    • NVIDIA GPU + Docker (for OCR) — optional, falls back to pdfplumber

    Backend¶

    cd vica
python -m venv .venv
.venv/Scripts/activate        # Windows
# source .venv/bin/activate   # Linux/macOS
pip install -r requirements.txt

# Pull the LLM model
ollama pull llama3.1:8b
ollama serve                   # keep running in separate terminal

# Start backend
python run_backend.py          # or: uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

    Frontend¶

    cd vica/frontend
npm install
npm run dev                    # Vite dev server on http://localhost:5173

    Vite proxies /api → http://localhost:8000.

    Environment Variables¶

    Copy .env.example to .env (optional — defaults work for local dev):

    LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=llama3.1:8b
LLM_ENABLED=true
OCR_BASE_URL=http://localhost:8001
OCR_ENABLED=true
SECRET_KEY=change-me
DEBUG=true
DOCKER_NETWORK=vica-network

    Docker Compose (Full GPU)¶

    Full stack with automatic GPU orchestration. Requires NVIDIA GPU + Docker with GPU support.

    Services¶

    Service Container Port Purpose
    Backend vica-backend 8000 FastAPI server
    Frontend vica-frontend 3000 React app via Nginx
    Ollama vica-ollama 11434 LLM server (always running)
    OCR vica-ocr 8001 vLLM OCR (managed by backend, not auto-started)

    First-time Setup¶

    cd vica

# Start services
docker compose up -d --build

# Pull the LLM model into the Ollama container
docker exec vica-ollama ollama pull llama3.1:8b
# Or use the setup script:
# bash scripts/setup-ollama-model.sh

    Usage¶

    docker compose up -d              # Start
docker compose up -d --build      # Rebuild and start
docker compose logs -f backend    # Watch backend logs
docker compose down               # Stop everything

    Key Configuration¶

    The OCR container (vica-ocr) is defined in compose but not auto-started — it uses the ocr-manual profile. The backend starts/stops it dynamically via the Docker socket when OCR is needed.

    # Backend needs Docker socket to manage OCR container
volumes:
  - /var/run/docker.sock:/var/run/docker.sock

    Environment overrides inside Docker: - LLM_BASE_URL=http://ollama:11434/v1 (Docker service name, not localhost) - OCR_BASE_URL=http://host.docker.internal:8001 (OCR container exposes port to host)

    Volumes¶

    Volume Purpose
    ./data Persistent data (CVs, JSON DB, reports, taxonomy)
    vica-ollama-models Ollama downloaded models
    vica-vllm-cache vLLM model cache

    Docker Compose (Portable / CPU-only)¶

    No GPU, no Ollama, no vLLM. Uses deterministic fallbacks: - OCR → pdfplumber (text-based PDF extraction only) - Skill extraction → phrase matching (taxonomy keyword matching) - Analysis → binary presence mapping (no semantic analysis)

    Use case: Viewing past run results on a laptop, lightweight testing, demo environments.

    docker compose -f docker-compose.portable.yml up -d --build
docker compose -f docker-compose.portable.yml down

    Only two services: backend (port 8000) + frontend (port 3000).

    Environment: LLM_ENABLED=false, OCR_ENABLED=false.

    Data Directory¶

    All persistent data lives in data/:

    data/
├── db/              # JSON persistence (candidates, jobs, skills, matches, etc.)
├── files/           # Uploaded CV files
│   └── cvs/         # CVs stored as {uuid}_{filename}
├── taxonomy/        # ESCO skills taxonomy (esco_skills.json)
├── skill_library/   # Cached skill phrases
└── reports/         # Generated HTML reports (if saved)

    Portability: Copy the entire data/ directory to move between machines. The portable Docker setup reads the same data/ structure.

    Nginx Configuration¶

    The frontend Dockerfile builds the React app and serves it via Nginx. nginx.conf proxies /api requests to the backend:

    location /api/ {
    proxy_pass http://backend:8000/;
}

    In production, the frontend is served on port 3000 via Nginx. In development, Vite's built-in proxy handles the same routing.

    Test Run¶

    Verify the full pipeline works:

    # Without LLM (phrase matching, fast)
python scripts/run_test.py

# With LLM (requires Ollama running)
python scripts/run_test.py --llm

    Seeds 100 fake CVs + 1 tech job, runs SPEA2, generates a report at data/test_report.html.
